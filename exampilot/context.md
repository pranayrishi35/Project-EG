# ExamPilot: Project Handoff & Context Document

## Overview
**ExamPilot** is an AI-powered, offline-capable Study Planner designed to help students prepare for competitive exams. By uploading a syllabus (PDF/Image) or providing a text description, students receive a personalized, day-by-day study schedule generated by Google's Gemini AI. The application features a premium "SaaS" aesthetic with gamification elements to maximize student retention.

## Technology Stack
- **Framework:** Next.js 14 (App Router)
- **Styling:** Tailwind CSS (Strictly native utility classes, no external UI libraries like Shadcn/MUI)
- **Database & Auth:** Supabase (PostgreSQL + Auth SSR)
- **AI Engine:** Google Generative AI (`@google/generative-ai` using Gemini 1.5 Flash)
- **PWA:** `next-pwa` (Service workers, manifest, installable on mobile)

## Core Features & Implementation Details

### 1. Database Schema
We rely heavily on Supabase's `JSONB` capabilities to avoid complex SQL migrations. 
- **Table:** `study_plans`
- **Columns:** `id` (UUID), `user_id` (UUID), `created_at` (Timestamp), `exam_name` (Text), `exam_date` (Date), `syllabus_text` (Text), `generated_plan` (JSONB).
- **JSONB Strategy:** The `generated_plan` column stores the AI output (Weeks, Days, Topics). We've dynamically extended this JSONB object to also store state:
  - `completed_topics`: An array of strings tracking checked-off tasks (e.g., `"w1d3t0"`).
  - `mock_tests`: An array of objects logging the user's mock test results.

### 2. Home Dashboard (`/`)
- A premium "Bento Box" grid layout.
- Fetches the user's Daily Streak and Top 2 recent active plans server-side.
- Left column embeds the `CreatePlanForm` in a `compact` mode.
- Right column provides Quick Actions (My Planner, Mock Test Analytics, Daily Flashcards [Coming Soon], Settings) and mini-cards for active plans.

### 3. Study Plan Generation (`/actions/planner.ts`)
- Uses Server Actions and `useTransition` for a seamless loading state.
- Accepts a `FormData` object containing the Exam Name, Date, and an optional Syllabus File (PDF/Image).
- Sends a structured prompt to Gemini 1.5 Flash to enforce a strict JSON output matching our `GeneratedPlan` type.
- Saves the resulting JSON into the Supabase `study_plans` table and redirects the user.

### 4. Interactive Plan Viewer (`/planner/[id]`)
- **Study Schedule Tab:** Displays the day-by-day plan. Users can tap topics to mark them complete (optimistic UI update + background Supabase JSONB sync).
- **Exam Pressure Mode:** A sleek toggle that dims the background, hides navigation, and applies a distraction-free slate-900 overlay to keep the student deeply focused.
- **Cheat Sheet Export:** A custom `@media print` CSS configuration that strips the UI and reformats the schedule into a dense, 2-column, ink-friendly physical cheat sheet triggered via `window.print()`.
- **Pomodoro Focus Timer:** A floating bottom-right widget to track study sessions.

### 5. Mock Test Analyzer (`MockTestAnalyzer.tsx`)
- Allows students to log their mock test scores.
- Supports custom grading schemes (e.g., +3 for correct, -1 for incorrect).
- Form calculates the live score and accuracy derived entirely from inputs (no unnecessary `useState`).
- Features a **pure Tailwind CSS Trend Chart** (bar chart) showing accuracy history and a colour-coded grading scale (Green >= 80%, Amber >= 60%, Red < 60%).
- Data is safely appended to the `mock_tests` array inside the existing `generated_plan` JSONB column.

### 6. PWA & Mobile Responsiveness
- Designed with mobile-first principles. 
- Features a `BottomNav` for mobile and standard `Header` for desktop/tablet. Both are hidden during Cheat Sheet printing via `print:hidden`.

## Development Rules & Constraints
1. **Design:** Keep it extremely minimal, clean, and fast. Prioritize the Indigo/Violet/Slate aesthetic. 
2. **Dependencies:** Avoid external UI/Chart libraries. Build visualizers (like the Mock Test trend chart) using pure Tailwind/CSS.
3. **Fonts:** No `next/font/google` due to constraints; strictly rely on local system fonts.
4. **React Patterns:** Next.js Server Components by default. Use Client Components specifically for interactivity (`useState`, `useTransition`). No React 19 hooks.
5. **Middleware:** `export const runtime = "nodejs"` is strictly mandatory in `middleware.ts`.
